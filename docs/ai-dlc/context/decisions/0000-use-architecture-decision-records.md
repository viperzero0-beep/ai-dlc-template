# 0000. ADR を採用する

- ステータス: 承認済み
- 日付: 2025-01-01
- 決定者: プロジェクトチーム

## 背景・課題

AI-DLC では AI が多くの設計案を高速に提案します。意思決定の**理由**を残さないと、
後から「なぜこの選択をしたか」が失われ、AI も人間も同じ議論を繰り返してしまいます。

## 決定

重要な技術的意思決定は ADR（Architecture Decision Record）として
`docs/ai-dlc/context/decisions/` に連番の Markdown で記録する。

## 理由

- 永続コンテキストとして次フェーズ・次セッションの AI に文脈を渡せる。
- レビューや振り返りで判断の経緯を追える。
- 軽量（1 ファイル）で、コードと一緒にバージョン管理できる。

## トレードオフ・影響

- 記録の手間が少し増える（`/adr` で自動化して軽減）。
- 記録しないと永続コンテキストの価値が下がるため、PR チェックリストで促す。

## 検討した代替案

- **Wiki / 外部ツール**: リポジトリと乖離しやすく、AI からも参照しづらいため却下。
- **記録しない**: 文脈喪失のリスクが高く却下。
