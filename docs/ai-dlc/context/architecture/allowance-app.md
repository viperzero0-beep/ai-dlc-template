# 小遣い帳アプリ 設計ドキュメント

## 1. 概要

個人が日々の小遣いを手軽に記録・管理できるシンプルな Web アプリです。ブラウザのローカルストレージを使用しており、ログイン不要で、開いたその場で利用できます。

## 3. データモデル

### Transaction
```javascript
{
  id: string,              // 一意識別子（日付-ランダム値）
  date: string,            // ISO 8601 形式（YYYY-MM-DD）
  type: 'income' | 'expense',  // 収入か支出か
  amount: number,          // 金額（整数）
  category: string,        // カテゴリ名（例: 食費、おこづかい）
  memo: string             // メモ（オプション）
}
```

### 永続化
- **保存先**: ブラウザの localStorage
- **キー**: `allowance-app-transactions`
- **形式**: JSON 配列

## 4. 実装した機能

### 4.1 Core Features

#### 取引登録
- 日付、種類（収入/支出）、金額、カテゴリ、メモを入力
- 保存後、自動的にローカルストレージに保存
- 新規入力後、フォームをリセット

#### 残高・集計表示
- 全期間での収入・支出・残高を表示
- 月次集計（選択月の合計金額）
- 最新順でソート

#### 取引一覧表示
- 月ごとの絞り込み可能
- 日付、カテゴリ、メモ、金額を表示
- 収入は緑、支出は赤でビジュアル区別

#### 削除機能
- 各取引の右側に削除ボタン
- クリック時に確認ダイアログを表示
- 削除後、自動更新

#### カテゴリ別集計（タブ機能）
- 「取引一覧」「カテゴリ別集計」の 2 タブ
- カテゴリごとに収入・支出・差額を表示
- 月ごとに絞り込み可能

### 4.2 非機能要件の実現

| 要件                 | 実装方法                                         |
| -------------------- | ------------------------------------------------ |
| 直感的な使いやすさ   | シンプルなフォーム、明確なラベル、視覚的な色分け |
| モバイル対応         | メディアクエリで 700px 以下で縦積みレイアウト    |
| 送信内容の保存確実性 | JSON.stringify/parse で確実に保存・復元          |
| 最小限の画面構成     | 登録・集計・一覧を 1 ページにまとめた            |

## 5. 技術スタック

| レイヤー   | 技術                | 理由                       |
| ---------- | ------------------- | -------------------------- |
| HTML       | 標準 HTML5          | 依存なし、シンプル         |
| CSS        | 標準 CSS3           | Flexbox / Grid で十分      |
| JavaScript | 標準 ES6+           | 依存なし、小規模           |
| 保存       | localStorage        | 規模が小さく、サーバー不要 |
| テスト     | Node.js test runner | 依存なし、組み込み         |

## 7. 主要フロー

### ユーザー操作フロー
```
1. ユーザーがフォームに入力
   ↓
2. app.js が入力を検証
   ↓
3. allowance-app.js の createTransaction() でトランザクション生成
   ↓
4. localStorage に保存
   ↓
5. render() で UI を再描画
   ↓
6. 一覧・集計・月次が更新
```

### タブ切り替え
```
1. ユーザーがタブボタンをクリック
   ↓
2. 現在のタブ/コンテンツから active クラスを除去
   ↓
3. 選択タブに active クラスを追加
   ↓
4. CSS で表示/非表示を制御
```

## 8. 制約・トレードオフ

| 項目           | 制約         | 理由                   |
| -------------- | ------------ | ---------------------- |
| ユーザー認証   | なし         | MVP スコープ外         |
| データ同期     | なし         | ローカルストレージのみ |
| バックアップ   | なし         | 手動 export を想定     |
| 編集機能       | 削除のみ実装 | 使用例が限定的         |
| グラフ表示     | なし         | MVP スコープ外         |
| オフライン対応 | 完全対応     | localStorage ベース    |

## 10. パフォーマンス特性

- **初期読み込み**: localStorage からの復元は即座
- **取引追加**: 数 ms（localStorage 操作）
- **月別集計**: O(n)（全トランザクション走査）
- **カテゴリ集計**: O(n)（全トランザクション走査）
- **スケーラビリティ**: 取引数が数千件程度なら問題なし

## 11. セキュリティ

- **入力検証**: HTML5 フォーム属性で基本的な検証
- **XSS 対策**: innerHTML で直接 HTML を設定せず、textContent で設定する部分もある（改善余地あり）
- **CSRF**: localStorage ベースなので CSRF リスク なし
- **データ保護**: 暗号化なし（ローカルストレージのため不要）

## 12. 既知の制限と今後の拡張

### 既知の制限
- データバックアップ機能がない
- 複数デバイス間の同期ができない
- 編集は削除→再入力のみ
- カテゴリの事前定義がない

### 今後の拡張案
1. **バックアップ・リストア** — JSON export/import
2. **複数ユーザー** — ログイン機能 + バックエンド
3. **インライン編集** — 取引の直接編集
4. **グラフ表示** — Chart.js など
5. **予算設定** — カテゴリ別の予算・アラート
6. **繰り返し取引** — 定期収入（給与など）の自動登録

## 13. 運用上の注意

- localStorage は同一オリジン内で共有される
- 別のタブでも同じデータにアクセス可能（リアルタイム同期なし）
- ブラウザのキャッシュ削除で全データが失われる
- 定期的な手動バックアップを推奨
